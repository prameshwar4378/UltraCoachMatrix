Dynamic Fee Categories Help

Dynamic Fee Categories are used to create reusable fee heads for the institute, so fees can be assigned, collected, tracked, and reported in a clear way.

  • Use this before fee collection when the institute wants separate heads such as Tuition Fee, Admission Fee, Exam Fee, Material Fee, Transport Fee, or Hostel Fee.
  • Default Amount saves a standard amount for the category, which helps staff add fees faster while still allowing changes where required.
  • Active categories should be used for current fee collection. Inactive categories are useful for old or discontinued fees that should not be selected anymore.
  • Invoice count helps identify categories already used in student invoices, so staff can avoid unnecessary deletion or renaming.
  • Reports and receipts become easier to understand because each payment is grouped under the correct fee category.
Important: create clear category names and avoid deleting categories already used in invoices, because they may be needed for old receipts, dues, and reports.

Dynamic Fee Categories institute साठी reusable fee heads तयार करण्यासाठी वापरले जातात, ज्यामुळे fees assign, collect, track आणि report करणे स्पष्ट होते.

  • Fee collection सुरू करण्यापूर्वी वापरा जेव्हा Tuition Fee, Admission Fee, Exam Fee, Material Fee, Transport Fee किंवा Hostel Fee सारखे वेगळे heads ठेवायचे असतात.
  • Default Amount category साठी standard amount ठेवतो, त्यामुळे staff fees जलद add करू शकतात आणि गरजेनुसार amount बदलू शकतात.
  • Active categories current fee collection साठी वापरा. जुने किंवा बंद झालेले fees select होऊ नयेत म्हणून त्यांना inactive ठेवता येते.
  • Invoice count category student invoices मध्ये वापरली आहे का हे दाखवतो, त्यामुळे delete किंवा rename करताना काळजी घेता येते.
  • Reports आणि receipts समजायला सोपे होतात कारण प्रत्येक payment योग्य fee category मध्ये group होते.
Important: category names स्पष्ट ठेवा आणि invoices मध्ये वापरलेल्या categories delete करू नका, कारण त्या old receipts, dues आणि reports साठी लागतात.

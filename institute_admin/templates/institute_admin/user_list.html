User Access Management Help

User Access Management is used to create and control login access for people who need to use the institute software.

  • Who should be added: admin staff, teachers, accountants, and student/parent users who need system access.
  • Why roles matter: each role opens only the work area needed for that person, such as teaching, fees, admin work, or student/parent access.
  • When to create access: create a user when a new staff member joins, a teacher needs login, an accountant handles fees, or a student/parent account is required.
  • When to update or deactivate: update role/contact details when responsibilities change, and deactivate/delete access when a person leaves or should no longer log in.
  • Use filters: search by role, status, name, username, phone, or email to quickly find the correct account.
Important: give users only the role they actually need. This keeps institute data safer and avoids accidental access to unrelated sections.

User Access Management चा वापर institute software वापरणाऱ्या लोकांसाठी login access तयार आणि control करण्यासाठी केला जातो.

  • कोणाला add करावे: admin staff, teachers, accountants आणि system access लागणारे student/parent users.
  • Role का महत्त्वाचा आहे: प्रत्येक role त्या व्यक्तीला लागणारेच work area उघडतो, जसे teaching, fees, admin work किंवा student/parent access.
  • Access कधी create करावा: नवीन staff join झाल्यावर, teacher ला login लागल्यावर, accountant fees handle करत असल्यावर किंवा student/parent account लागल्यावर.
  • Update किंवा deactivate कधी करावे: responsibility बदलल्यावर role/contact update करा, आणि व्यक्ती institute सोडल्यावर किंवा login नको असल्यावर access deactivate/delete करा.
  • Filters वापरा: role, status, name, username, phone किंवा email नुसार account पटकन शोधता येतो.
Important: user ला फक्त त्याला खरंच लागणारा role द्या. त्यामुळे institute data सुरक्षित राहतो आणि चुकीचा access टाळता येतो.

User Access Management का उपयोग institute software इस्तेमाल करने वाले लोगों के लिए login access बनाने और control करने के लिए किया जाता है.

  • किसे add करना चाहिए: admin staff, teachers, accountants और system access की जरूरत वाले student/parent users.
  • Role क्यों जरूरी है: हर role व्यक्ति को सिर्फ जरूरी work area देता है, जैसे teaching, fees, admin work या student/parent access.
  • Access कब create करें: नया staff join हो, teacher को login चाहिए, accountant fees handle करे, या student/parent account चाहिए.
  • Update या deactivate कब करें: responsibility बदलने पर role/contact update करें, और व्यक्ति institute छोड़ दे या login access नहीं चाहिए तो access deactivate/delete करें.
  • Filters इस्तेमाल करें: role, status, name, username, phone या email से account जल्दी खोजा जा सकता है.
Important: user को केवल वही role दें जिसकी उसे सच में जरूरत है. इससे institute data सुरक्षित रहता है और गलत access से बचाव होता है.
